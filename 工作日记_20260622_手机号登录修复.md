# 工作日记 - 2026年6月22日

## 任务：修复微信小程序手机号登录"访问超时"Bug

---

## 一、问题现象

- 点击「获取手机号」授权按钮后，始终弹出"访问超时，请重试!"
- 控制台 `getPhoneNumber:ok`，说明微信端授权成功
- 后端返回 `{"code":9999,"msg":"异常:JSONObject[\"session_key\"] not found."}`

---

## 二、根因分析（共 4 个 Bug）

### Bug 1：`wx.login()` 重复调用导致 code 不匹配

**位置**：`getPhoneNumber` 回调内又调了 `wx.login()`

**原因**：`wx.login()` 生成了新的 code，对应新的 session_key。但 `encryptedData` 是用旧的 session_key 加密的，后端拿新 session_key 解不开。

### Bug 2：content-type 导致 base64 损坏

**位置**：请求 header 用了 `application/x-www-form-urlencoded`

**原因**：`encryptedData` 中的 `+` 号（如 `C+40`、`ART+3RS`）被 form-urlencoded 转成空格，AES 解密必然失败。

### Bug 3：code 类型错误（根因）

**位置**：前端用了 `e.detail.code`（`getPhoneNumber` 回调里的 code）

**原因**：后端 `QGQCPhoneServlet` 调的是微信 `jscode2session` 接口，这个接口需要 `wx.login()` 类型的 code。`getPhoneNumber` 回调里的 code 是另一种类型，微信不认识，返回 `session_key not found`。

### Bug 4：调错了后端接口

**位置**：前端调的是旧接口 `PhoneServlet`，不是能正常工作的新接口 `QGQCPhoneServlet`

**原因**：`app.globalData.allUrl.getPhone` → `/register_war/servlet/PhoneServlet`（旧，可能已废弃），正确接口是 `app.globalData.allUrl.QGQCPhoneServlet` → `/evaluation/QGQCPhoneServlet`（新，Spring Boot）。

---

## 三、最终方案

```
页面加载 → wx.login() → code 缓存到 _phoneLoginCode（不消费！）
用户点按钮 → getPhoneNumber → {encryptedData, iv}
前端发请求 → POST /evaluation/QGQCPhoneServlet {code, encryptedData, iv}
后端 → jscode2session(code) → session_key → AES解密 → phoneNumber ✅
```

关键：**`wx.login()` 必须在点按钮之前调用，且拿到的 code 不能用于其他任何事情。**

## 六、关于"不关注切瓜切菜公众号就卡住"的问题

这是**微信开发者工具的限制**，不是代码 bug。

原因：开发工具无法模拟真实微信的公众号关注状态，当代码调用 `https://mb.fsmbdlkj.com/wxpublic/WXPublic/getList` 查询是否关注时，可能：
1. 返回空数据，代码路径走向未关注分支
2. 某些页面会弹出"前往关注"弹窗，阻止后续操作

**解决方案**：
- **真机预览**：在开发者工具点击「预览」，用手机微信扫码，真机上可以正常关注/跳过
- **上传体验版**后扫码测试，体验版不会被卡住
- 已在 `pages/index/index.js:1097` 和 `pages/main/main.js:2235` 看到注释 `//去掉关注公众号的逻辑`，说明部分页面已经绕过了这个检查
