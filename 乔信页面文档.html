chatWithCustDetail — 聊天详情页功能

  • 进入页面调 getOldData()，获取历史消息（分页，每页 20 条，向上滚动加载更多）
  • 发送文字：输入框回车或点发送 → sendMsgExec(1, msg)
  • 发送图片：chooseMedia → 上传到 OSS → sendMsgExec(2, url)
  • 发送视频：chooseMedia → 上传到 OSS → sendMsgExec(4, url)
  • 发送语音：长按录音（最短 2 秒）→ 上传到 OSS → sendMsgExec(3, url, duration)
  • 发送后调 getNewData() 刷新，同时通过 RabbitMQ 推送通知对方
  • 支持预览大图（wx.previewImage）、播放语音（InnerAudioContext）

消息类型（msg_type）

值类型
1文字
2图片
3语音（含 duration 字段，单位秒）
4视频

wechat_type（聊天类型）

值说明
1店员 → 客户
2客户 → 店员
3客户 → 客户（乔信用此类型）

chatWithCust — 会话列表页

方法路径参数说明
GET /evaluation/selectWeChatMsgRecord_QGQC_Customer shop_id, bind_person_id 初始化会话列表，返回所有会话及最新一条消息、未读数
GET /evaluation/selectWechatMsgRecord_Last wechat_type, sender_id, receiver_id 获取某对话最新一条消息（从 chatWithCustDetail 返回时调用，刷新列表）
POST /evaluation/QGQCPhoneServlet code, encryptedData, iv, wechatAppId "我"Tab：获取手机号
POST updateCustomerNameByOpenId（app.globalData 动态） openid, name "我"Tab：保存昵称
POST updateCustomerPhoneByOpenId（app.globalData 动态） openid, phone "我"Tab：保存手机号
POST updateCustomerAvatarUrlByOpenId（app.globalData 动态） openid, avatarUrl "我"Tab：保存头像
UPLOAD UploadAliYunFile（app.globalData 动态） file（multipart） "我"Tab：头像上传到阿里云 OSS

chatWithCustDetail — 聊天详情页

方法路径参数说明
GET /evaluation/selectWechatMsgRecord_StaffToCustomer_Old max_id, page_size, customer_id, shop_id, staff_id 拉取历史消息（向上滚动加载），max_id=0 时服务器用 int 最大值，即全量最新
GET /evaluation/selectWechatMsgRecord_StaffToCustomer_New min_id, page_size, customer_id, shop_id, staff_id 拉取新消息（发送后刷新）
POST /evaluation/insertWechatMsgRecord shop_id, msg_type, msg, duration, wechat_type, sender_id, receiver_id 发送消息（文字/图片/语音/视频），wechat_type=3
POST /evaluation/SendRabbitMqMsg task_queue_names（格式：hj + "_qgqc_" + receiver_id）, msg（JSON） 发送成功后推送 RabbitMQ 通知，触发对方实时收消息
UPLOAD UploadFile（app.globalData 动态） file（multipart），formData: oldUrl 图片 / 视频 / 语音上传，返回 object 字段为 OSS URL，然后再调 insertWechatMsgRecord

关键数据结构

会话列表项（selectWeChatMsgRecord_QGQC_Customer 返回）

字段说明
customer_id对方客户 ID
customer_name对方昵称，值为"未知"时不显示
avatar_url对方头像
lstWechatMsgRecord数组，取 [0] 显示最后一条消息，含 msg、is_not_read_count、create_time
time前端格式化后的时间字符串（weChatTimeFormat）

消息记录项（recordList 每条）

字段说明
id消息 ID，用于分页 max_id / min_id 和 scroll-into-view（page_{{id}}）
sender_id发送者 ID，与 customerInf.id 比较判断左右气泡
msg_type1 文字 / 2 图片 / 3 语音 / 4 视频
msg消息内容（文字）或 OSS URL（图片/视频/语音）
duration语音时长（秒），气泡宽度 = (duration%61+5)*2 rpx

实时消息机制

乔信不用轮询，走 RabbitMQ + 小程序 WebSocket（app.js 全局监听）。

  • 发送消息后调 SendRabbitMqMsg，队列名：{hj}_qgqc_{receiver_id}
  • app.js 全局收到推送后，找到当前页面（chatWithCust 或 chatWithCustDetail）调对应刷新方法
  • chatWithCust 收到推送 → 调 selectWechatMsgRecord_Last 刷新列表最后一条
  • chatWithCustDetail 收到推送 → 调 getNewData 拉取新消息
